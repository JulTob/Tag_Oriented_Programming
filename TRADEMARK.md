- Implement the TOP paradigm in any language.
- Build libraries, tools, and products on it, including commercially.
- Say your work *implements* or *is inspired by* Tag-Oriented Programming, with attribution.
- Use the names nominatively to refer to the paradigm and this project.

## Please do not

- Use the names (or a confusingly similar one) as the name of *your* product in a way that implies it is the official project or is endorsed by it.
- Use "**TOP Verified**" / "TOP-conformant" for your implementation **without the steward's authorization** (see [`CONFORMANCE.md`](CONFORMANCE.md)).
- Modify the Specification and redistribute it under the same name as if it were the official text.

The intent is **not** to restrict the ideas — they are open, and implementations are encouraged — but to keep one trustworthy source of truth, so a sloppy fork can't be mistaken for the standard.
